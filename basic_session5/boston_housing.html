编程练习3：定义衡量标准¶

如果不能对模型的训练和测试的表现进行量化地评估，我们就很难衡量模型的好坏。通常我们会定义一些衡量标准，这些标准可以通过对某些误差或者拟合程度的计算来得到。在这个项目中，你将通过运算决定系数 R2 来量化模型的表现。模型的决定系数是回归分析中十分常用的统计信息，经常被当作衡量模型预测能力好坏的标准。

R2的数值范围从0至1，表示目标变量的预测值和实际值之间的相关程度平方的百分比。一个模型的R2 值为0还不如直接用平均值来预测效果好；而一个R2 值为1的模型则可以对目标变量进行完美的预测。从0至1之间的数值，则表示该模型中目标变量中有百分之多少能够用特征来解释。模型也可能出现负值的R2，这种情况下模型所做预测有时会比直接计算目标变量的平均值差很多。

在下方代码的 performance_metric 函数中，你要实现：

  • 使用 sklearn.metrics 中的 r2_score 来计算 y_true 和 y_predict的R2值，作为对其表现的评判。
  • 将他们的表现评分储存到score变量中。

或

  • (可选) 不使用任何外部库，参考决定系数的定义进行计算，这也可以帮助你更好的理解决定系数在什么情况下等于0或等于1。

from sklearn.metrics import r2_score

def performance_metric(y_true, y_predict):
    """计算并返回预测值相比于预测值的分数"""
    
    score = r2_score(y_true, y_predict)

    return score

def performance_metric2(y_true, y_predict):
    """计算并返回预测值相比于预测值的分数"""
    y_bar = sum(y_true)/len(y_true)
    
    SS_tot = sum([(y-y_bar)**2 for y in y_true])
    
    SS_res = sum([(y-f)**2 for y,f in zip(y_true,y_predict)])
    
    score = 1-SS_res/SS_tot

    return score

问题 3 - 回答:¶

拟合度R2的取值范围为[0,1]。R2的值越接近1，说明模型对训练数据的拟合程度越好；反之，R2的值越接近0，说明模型对训练数据的拟合程度越差。

当然是否满足要求（成功的描述目标变量的变化）还取决于我们对模型的精度要求。如果我们期望更高精度的训练结果，比如拟合度大于0.975，那么该结果是不满足要求的，不能算作成功。

另一方面，这个测试仅仅使用了5个数据。这5个数据能否有效代表整体是一个未知数。如果整体数据量很大，那么仅从中挑出5个数据来进行验证，这个验证结果会很不稳定，也不可信。这就是要做K折交叉检验的必要性。

问题 7- 网格搜索（Grid Search）¶

什么是网格搜索法？如何用它来优化模型？

问题 7 - 回答:¶

网格搜索法简单来说就是用穷举法测试给定参数取值范围下的所有组合，然后通过给定的评价函数找到效果最好的那组参数。

所谓网格，可以理解为给定参数组合矩阵。例如有两个参数A和B，参数A的取值范围为[0,5]区间内的整数，参数B的取值范围为(10,20,30)，则可以构建如下形式的参数网格：

0 1 2 3 4 5
10 (0,10) (1,10) (2,10) (3,10) (4,10) (5,10)
20 (0,20) (1,20) (2,20) (3,20) (4,20) (5,20)
30 (0,30) (1,30) (2,30) (3,30) (4,30) (5,30)

其中构建网格的参数是我们给定的，参数的取值范围也是给给定的。

为了确定那组参数是最优的，需要给定评价函数用于参数效果评估。

问题 8 - 回答：¶

  1. K折交叉验证，首先将全体数据分割成K个子集，其中一个子集作为验证数据集，其他K-1个子集用来训练，训练完成后用验证数据集进行验证。重复K次上述过程，每个子子集验证一次，平均K次的结果，最终得到一个最终的结果。默认情况下 Kfold 是对数据按顺序切分，即每次切分的数据切分的结果都一样。可以通过shuffle=True设置为随机切分。
  2. GridSearchCV会将每一个参数组合，都利用交叉检验的方式执行一遍，即在训练集和测试集上分别运行计算出训练分和测试分，最终得到每个参数组合的平均分，通过对分值的比较，找到最优的参数组合。
  3. 'cv_results_'能够提供很多信息，除了运行时间，还包括每个参数在每个训练集和测试集上的训练分数和测试分数，以及平均训练分数和测试分数。最终告诉我们哪个参数是最有参数。
  4. 如果不使用交叉验证可能出现过拟合或欠拟合的情况。由于数据本身可能存在一定的顺序上的关联性，比如某一类数据都集中在数据集的开头，如果固定划分训练集和测试集，可能使训练集或测试集数据带有了某类较强的特征属性而缺少了其他属性，无法很好地泛化或验证。采用交叉验证，重复划分组合出不同的训练集和测试集，从有限的数据中尽可能获得多的有效信息，反复训练、测试，最终返回模型准确率的均值，从而避免过拟合和欠拟合。

在这个练习中，你将需要将所学到的内容整合，使用决策树算法训练一个模型。为了得出的是一个最优模型，你需要使用网格搜索法训练模型，以找到最佳的 'max_depth' 参数。你可以把'max_depth' 参数理解为决策树算法在做出预测前，允许其对数据提出问题的数量。决策树是监督学习算法中的一种。

在下方 fit_model 函数中，你需要做的是：

  1. 定义 'cross_validator' 变量: 使用 sklearn.model_selection 中的 KFold 创建一个交叉验证生成器对象;

  3. 定义 'params' 变量: 为 'max_depth' 参数创造一个字典，它的值是从1至10的数组;
  4. 定义 'scoring_fnc' 变量: 使用 sklearn.metrics 中的 make_scorer 创建一个评分函数； 将 ‘performance_metric’ 作为参数传至这个函数中；
  5. 定义 'grid' 变量: 使用 sklearn.model_selection 中的 GridSearchCV 创建一个网格搜索对象；将变量'regressor', 'params', 'scoring_fnc'和 'cross_validator' 作为参数传至这个对象构造函数中；

def splitData(df, k):
    """ 将数据集分割成k个子集 """
    n = len(df)
    if n < k: #如果数据数小于子集数，抛出异常
        raise Exception
        
    df_list = []
    num = n/k
    mod = n%k
    # 平分为k个子集
    for i in range(k):
        start = num*i
        end = num*(i+1)
        df_list.append(df[start:end])
        
    # 无法整除的部分全部整合到最有一个子集中    
    if mod != 0:
        last_df = df_list.pop()
        df_list.append(pd.concat([last_df,df[-mod:]]))

    return df_list
    

def cross_val_score(estimator, X, y, scoring = performance_metric, cv=3):
    """ 返回每组交叉验证的模型分数的数组 """
    scores = [0]*cv
    X_list = splitData(X, cv)# 将特征切换分cv个子集
    y_list = splitData(y, cv)# 将结果切分为cv个子集
    # 遍历cv次，每次取一个子集做测试集，其余合并为训练集。交叉检验
    for i in range(cv):
        # 划分训练集和测试集
        X_test = X_list.pop()
        X_train = pd.concat(X_list)
        y_test = y_list.pop()
        y_train = pd.concat(y_list)
        # 训练
        model = estimator.fit(X_train, y_train)
        # 检验
        y_predict = model.predict(X_test)
        score = performance_metric(y_test, y_predict)
        scores[i] = score
        # 测试集重新加入列表头
        X_list.insert(0, X_test)
        y_list.insert(0, y_test)
    
    return scores

def fit_model2(X, y):
    """ 基于输入数据 [X,y]，利于网格搜索找到最优的决策树模型"""
    
    #最优交叉验证分数对应的最优模型
    best_estimator = None
    max_score = 0
    for max_depth in range(1,11): # 最大深度1～10
        regressor = DecisionTreeRegressor(max_depth=max_depth)
        scores = cross_val_score(regressor, X, y)
        mean_score = sum(scores) / len(scores)
        if mean_score > max_score:
            max_score = mean_score
            best_estimator = regressor
    
    return best_estimator

问题 13 - 回答：¶

  1. 1978年的数据确实比较久远，距今已有近40年之久，即便是已考虑了通货膨胀也无法在今天适用。首先随着时间的增长，考虑进通胀的偏差会越来越大；其次房价的增长与通胀并不同步，通胀一般指CPI（消费物价指数），而房产作为资产，房价的增长并不会计入通胀；最后影响房价的因素很多，整体宏观环境因素我们并没有考虑进来，比如“次贷危机”等。
  2. 数据中呈现的特征也并不足以描述一个房屋。影响房价的因素非常多，以中国市场为例，除了面积和户型等因素，还有是否是“学区房”？交通是否便利？以及楼龄等诸多因素。
  3. 大数据采集的数据是很难用于其他乡镇地区的。就如同在中国，从北上广深一线城市采集的房价是不适用于三四线城市的。
  4. 如第2问所讲，社区环境会在一定程度上影响房价，但是仅凭社区环境来判断房价是片面的，不合理的。
